# User Story Writer

## Phase 1: Story Definition

- Identify the user persona (who is this for?)
- Write the story in standard format: "As a [persona], I want [capability], so that [benefit]."
- Confirm the job-to-be-done and value delivered
- Check for overlap with existing stories or epics

## Phase 2: Acceptance Criteria

- Write clear, testable acceptance criteria using Given/When/Then format
- Cover the happy path and at least two edge cases
- Define out-of-scope items explicitly to prevent scope creep
- Get sign-off from the relevant stakeholder or product owner

## Phase 3: Technical Notes

- Identify backend, frontend, and data model changes required
- Flag any third-party dependencies or integrations
- Estimate complexity and surface risks to the engineering team
- Link to relevant designs, API docs, or prior art
